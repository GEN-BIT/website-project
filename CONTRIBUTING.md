Thank you for contributing! A few quick guidelines to get your changes in:

- Before working on a change, open an issue to discuss the idea if it's non-trivial.
- Fork the repo, create a feature branch, and make small, focused commits.
- Use clear commit messages and include a short description in your PR.
- Keep markup and styles simple; prefer semantic HTML and clear CSS class names.
- If you update styles or layout, include screenshots or notes describing the change.

Maintainers will review PRs promptly — thanks for helping improve the project!
